# 修復成果：馬克筆透明度與重設同步功能

此次更新解決了馬克筆（Marker）與螢光筆（Highlighter）的視覺效果問題，並修復了「重設標記 (Reset Markup)」在多裝置間無法同步清除的 Bug。

## 變更摘要

### 1. 渲染性能與品質修復 (馬克筆 & 螢光筆)
- **修正雙重繪製 Bug**：修正了 `AnnotationRenderer.js` 導致筆觸重複繪製兩次的邏輯錯誤。原本會讓 60% 的透明度疊加到約 84%，修正後顏色恢復通透。
- **馬克筆 (Marker) 優化**：由原本的 100% 實心改為 **75% 半透明 (Alpha: `BF`)**。
- **螢光筆 (Highlighter) 優化**：修正後其 **60% 透明度** 配合 `Multiply` 混合模式呈現更真實的效果。

### 2. 同步機制修復 (重設標記功能)
- **墓碑 (Tombstone) 刪除傳遞**：修復了當 Machine A 執行「Library -> Current -> Reset Markup & Stats」時，Machine B 不會跟著同步清除的問題。
- **實作原理**：重設時不再只是單純刪除本機資料，而是為所有舊標記生成 `deleted: true` 的「墓碑」記錄並推送到雲端。
- **自動清除**：Machine B 接收到這些墓碑訊號後，會主動清除當前樂譜的本機快取，確保兩端狀態達成 100% 一致。

## 驗證結果

| 功能 | 測試情境 | 效果描述 |
| :--- | :--- | :--- |
| **透明度** | 馬克筆與螢光筆塗抹於音符上 | **清晰透徹**，音符不會被墨水完全遮蓋。 |
| **重設同步** | Machine A 重設 -> 檢查 Machine B | **即時清空**，標記與統計數據在兩端同步消失。 |
| **穩定性** | 頻繁重設與再次標記 | 墓碑機制有效防止舊標記「死而復生」。 |
