# 小節數工具 (Measure Numbering Tool) 設計文件

## 1. 概述
小節數工具是 ScoreFlow 中專為音樂家設計的功能，用於在樂譜上快速標記小節序號。系統支援**多樂章/多曲目**層級標註，並具備**頁面感知對齊**技術，確保小節號在視覺上與譜面結構完美融合。

## 2. 核心模式設計

### A. 連續小節模式 (`measure`)
*   **定位邏輯：頁面自動磁吸 (Smart Snap)**
    *   **首個對齊**：每一頁放置的第一個小節號決定該頁的基準 X 軸坐標。
    *   **後續磁吸**：同頁面後續放置的小節號會自動對齊該頁第一個小節的 X 坐標，確保整頁垂直排列整齊。
*   **適用場景**：建立樂譜基礎架構，精確對齊每行譜表的開頭。
*   **視覺樣式**：淺色外框矩形，半透明文字。

### B. 自由小節模式 (`measure-free`)
*   **定位邏輯**：完全自由定位，不觸發對齊。
*   **適用場景**：標註臨時練習點、多聲部中的特定小節、或不規則排版位置。
*   **視覺樣式**：圓形外框 (Circle Frame)，強調標註感。

## 3. UI Layout (介面佈局)

### 3.1 數字鍵盤視窗 (`#measure-dialog`)
採用系統統一的 **Glassmorphism (毛玻璃)** 視覺風格，設計為懸浮中央的 Modal 視窗。

*   **視覺風格**：
    *   **背景**：`-webkit-backdrop-filter: blur(8px)`，半透明深色遮罩。
    *   **寬度**：固定 `380px` (桌面) / `90vw` (行動裝置)。
    *   **圓角**：`20px` 圓角與柔和陰影。

### 3.2 輸入區域配置 (Input Display Area)
視窗頂部為水平排列的雙欄位輸入區：
*   **左欄：段落名稱或曲目順序** (`#measure-song-display`)
    *   **內容**：填寫「第一樂章」、「第二首」等資訊。
    *   **互動**：點擊可切換為活動欄位（文字高亮顯示為主題色）。
*   **中央：分隔符**
    *   顯示為中性的橫槓 (`-`) 或空格，僅供視覺區隔。
*   **右欄：小節數字欄位** (`#measure-num-display`)
    *   **內容**：顯示主要小節序號（如 15, 135）。
    *   **字體**：加大字級 (`36px`)，方便觸控識別。

### 3.3 步進控制 (Step Controls)
位於輸入區下方，用於快速調整自動遞增量：
*   **樣式**：包含 `+` 與 `-` 按鈕以及中央的數值顯示。
*   **預設值**：`4` (可根據需求動態調整)。

### 3.4 數字鍵盤網格 (Keypad Grid)
*   **佈置**：`3 x 4` 的格狀按鈕。
*   **特殊功能鍵**：
    *   **0-9**：數字輸入。
    *   **Del (退格鍵)**：刪除最後一個字元。
    *   **Clear (清除)**：快速清空目前活動欄位。
    *   **確定 (OK)**：儲存並關閉視窗。

## 4. 技術實作細節

### 4.1 數據結構 (Data Schema)
```javascript
{
    type: 'measure' | 'measure-free',
    page: 1,
    x: 0.08,             // 根據該頁首個小節自動設定
    y: 0.25,
    data: "Mov.I - 15",  // 儲存格式："段落名稱或曲目順序 - 小節序號"
    variant: 'measure'
}
```

### 4.2 頁面感知磁吸邏輯
在 `InteractionManager.js` 中，系統會檢查目前頁面是否已存在小節標註。若存在，新標註會直接繼承既存標註的 `x` 坐標，達成視覺上的垂直對齊。

### 4.3 自動遞增與層級記憶
*   **雙欄位記憶**：系統會分別記憶「曲目順序」與「最後一個小節序號」。
*   **下一次點擊**：預設填入 `[記憶曲目] - [最後序號 + 步進]`。
