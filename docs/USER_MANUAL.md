## 1. 快速開始

### 開啟樂譜

1. 點擊首頁的 **Open Solo Score**，選取 PDF 檔案。
2. 樂譜載入後，工具列（右側浮動欄）與尺規（左側）自動出現。
3. 首次開啟時，系統在背景自動偵測五線譜行位置（見[第 4 節](#4-五線譜行偵測staff-detection)）。

### 基本瀏覽

| 操作 | 說明 |
|---|---|
| 捲動 / 滑動 | 連續瀏覽樂譜 |
| **Space** / **→** | Jump 前進（跳至下一段） |
| **←** | Jump 後退（回到上一個位置） |
| **W** | 適合寬度（Fit to Width） |
| **F** | 適合高度（Fit to Height） |

| 標記 | 顏色 | 說明 |
|---|---|---|
| 藍色短橫條 | 藍 `#3b82f6` | Anchor 錨點位置 |

## 3. Jump 導航系統

Jump 是 ScoreFlow 最核心的導航功能，設計給**演奏中**或**練習時**快速翻頁。

### 3.1 跳躍線（Jump Line）

視窗中有一條水平虛線，稱為**跳躍線**。它定義了視線焦點的位置——Jump 後，目標內容會對齊在這條線上。

- 預設位置：距視窗頂部 40px。
- 可拖曳調整位置（見[設定](#8-設定)）。

### 3.2 Jump 前進邏輯（按 Space 或 →）

系統依以下優先順序決定跳躍目標：

```
① Anchor 存在  →  跳至跳躍線下方最近的 Anchor
② System 存在  →  跳至「頁面重疊設定」指定的 System（見 3.3）
③ 最終 fallback  →  捲動 (viewport 高度 − 跳躍線偏移量)
```

### 3.3 頁面重疊（Page Overlap）

這是 System-based 跳躍的核心設定。

**概念：** 翻譜時，你眼前可以看到畫面中的多個 System（樂譜行）。按 Jump 後，你希望「保留倒數第 N 個 System 作為過渡銜接」——它會出現在新畫面的跳躍線位置。

**範例（預設 N = 1）：**

```
跳躍前：                    跳躍後：
┌─────────────────┐         ┌─────────────────┐
│  System A       │         │                 │
│ - - - - (線) - -│(跳躍線) │ - - - - (線) - -│(跳躍線)
│  System B       │         │  System D  ←────┤ 原畫面的最後一行
│  System C       │         │  System E       │
│  System D       │         │  System F       │
└─────────────────┘         └─────────────────┘
```

System D 是原畫面最後一行（倒數第 1 個），跳躍後它出現在新畫面的跳躍線上。

**設定位置：**

```
標記工具（T）→ 設定（齒輪圖示）→ More 分頁 → 頁面重疊 [−] N [+]
```

| 值 | 效果 |
|---|---|
| **1**（預設） | 最後一行 = 新畫面起點（最大跳躍距離） |
| **2** | 倒數第二行 = 新畫面起點（多保留一行上下文） |
| **3+** | 保留更多銜接上下文，跳躍距離更短 |

### 3.4 Jump 後退（按 ←）

每次 Jump 前進都會記錄位置到**歷史堆疊**（最多 50 筆）。按 ← 即精確回到上一個位置，與跳躍距離無關。

若歷史堆疊為空（如剛開啟樂譜後的第一次後退），會往上捲動一個等距。

## 4. 五線譜行偵測（Staff Detection）

Staff Detection 透過畫面像素分析，自動找出每一頁的樂譜行（System）位置，讓 Jump 在沒有手動 Anchor 的情況下也能精準導航。

### 4.1 自動偵測

開啟任何 **新** PDF 後，系統會在背景自動執行偵測，並顯示進度提示：

```
正在分析樂譜結構... 2 / 8
```

偵測完成後，結果儲存於本機（IndexedDB），下次開啟同一份 PDF 不會重複偵測。

> **支援掃描版 PDF：** 演算法採用自適應門檻，可處理泛黃、陰影等掃描背景。

### 4.2 手動重新偵測

若偵測結果不準確（例如特殊版面的樂譜），可手動重新執行：

```
標記工具（T）→ 設定 → More 分頁 → System Detection
```

| 按鈕 | 說明 |
|---|---|
| **Detect** | 清除舊結果，對目前 PDF 重新偵測全部頁面 |
| **刪除全部** | 清除所有 System 標記，Jump 退回 fallback 模式 |

偵測完成後，狀態文字會顯示 `已偵測 N 個 System`。

### 4.3 在尺規上顯示 System 位置

**System Detection** 區塊頂部有一個開關：

- **開啟** → 尺規上顯示淡藍色虛線，標示每個 System 的頂部位置。
- 其中**下一個 Jump 目標** System 以橘色顯示（與 Anchor next-target 視覺語言一致）。
- 預設關閉（不影響導航，純視覺輔助）。

### 4.4 偵測演算法概要

| 參數 | 說明 |
|---|---|
| 自適應背景門檻 | 取樣頁面六個邊緣區塊估算背景亮度，避免掃描版誤判 |
| 行密度門檻 | 超過 30% 橫向像素為深色時，認定為樂譜線 |
| 最大線條厚度 | 超過 8px 的深色帶視為文字或樑線，排除 |
| System 分群間距 | 線條群組間距超過 25px，認定為不同 System |

偵測結果以**比率（ratio）**儲存（System 頂部 Y / 頁面高度），與縮放比例無關。

## 5. Anchor 錨點

Anchor 是使用者手動放置的跳躍目標，優先順序高於 System。

### 5.1 放置 Anchor

1. 按 **A** 鍵，或點擊工具列的錨點按鈕，進入 Anchor 模式。
2. 點擊樂譜中的任意位置，放置錨點標記。
3. 再按 **A** 或 **V** 退出 Anchor 模式。

### 5.2 Anchor 的 Jump 行為

- **Space / →**：跳至跳躍線下方最近的 Anchor。尺規橘色標記指示下一個目標。
- **←**：回到上一個記錄位置（歷史堆疊）。

### 5.3 何時使用 Anchor vs System

| 情境 | 建議 |
|---|---|
| 一般練習 / 表演瀏覽 | 使用 System（自動，零設定） |
| 需要在特定段落精準停靠 | 放置 Anchor（手動，完全控制） |
| 混用 | Anchor 優先；Anchor 用完後自動切換至 System |

## 6. 標記與註記

### 6.1 開啟標記工具

按 **T** 或點擊工具列的 T 按鈕，開啟標記面板。

### 6.2 分類

標記分為以下類別（Categories）：

| 類別 | 說明 |
|---|---|
| Pens | 手繪線條、高亮 |
| Bowing | 弓法標記 |
| Fingering | 指法 |
| Articulation | 奏法記號 |
| Tempo | 速度標記 |
| Dynamic | 力度標記 |
| Anchor | 錨點 |

### 6.3 圖層（Layers）

每個類別可獨立顯示 / 隱藏：

```
標記工具（T）→ 設定 → Categories 分頁
```

### 6.4 編輯模式捲動條（View Scrolling Bar）

當您開啟標記面板進入編輯模式時，畫面右側會出現專屬的**視圖捲動條（Edit Scrollbar）**。
為了模擬真實紙本樂譜的操作體感，此捲動條採用**反向操作**設計：
- **向上拖曳**：將紙張往上拉，畫面會向**下**捲動（顯示樂譜下半部）。
- **向下拖曳**：將紙張往下拉，畫面會向**上**捲動（顯示樂譜上半部）。

此設計讓您在標記時，能有如同用手移動實體紙張般直覺的拖曳體驗。

### 6.5 橡皮擦工具

橡皮擦有兩種行為，視操作方式自動切換：

#### 整體清除（點擊）

**按一下**距離最近的物件 → 整個標記被刪除。適用所有標記類型（符號、文字、線條等）。

#### 分段清除（拖曳）

**按住並拖曳**橡皮擦橫跨筆畫 → 只移除被覆蓋到的那一段，兩端存活的部分保留為兩條獨立的筆畫。

適用於所有手繪線條類型：

| 類型 | 說明 |
|---|---|
| Pen（各色） | 一般鋼筆線條 |
| Highlighter（各色） | 高亮螢光標記 |
| Line | 直線 |
| Slur | 圓滑線 |

> 非線條類型的標記（符號、錨點、指法等）在拖曳時仍為整體刪除。

**範例：**

```
拖曳前：                拖曳後：
 ─────────────────      ────         ─────
        ↑ 橡皮擦路徑               (中段被切除)
```

#### 復原（Undo）

橡皮擦的每次拖曳操作（不論切了幾段）視為**單一動作**，按一次 **Ctrl+Z**（macOS: ⌘Z）即可完整復原，所有分段還原為原始筆畫。

---

### 6.6 修改已存在標記的樣式

如果您想更改已經畫好或放好的標記（如文字、線條、符號）的顏色、線條樣式或粗細，請依照以下步驟操作：

1. 點擊工具列的 **Select（箭頭）** 工具（或按鍵盤 `V` 鍵進入選取模式）。
2. 在畫面上點擊您想要修改的目標物件（被選取的物件周圍，會出現一圈虛線藍色光暈 🌟）。
3. 保持選取狀態，直接點開工具列上的 **「三個點（選項面板）」**。
4. 點選您想要的 **新顏色**、**線條樣式** 或 **粗細 (Size)**。
5. 物件的樣式會立刻更新並自動存檔。

## 8. 設定

### 8.1 全域設定（Global Settings）

點擊側邊欄頂部的齒輪圖示開啟，包含三個分頁：

| 分頁 | 內容 |
|---|---|
| Cloud & Account | Google Drive 同步設定 |
| System Settings | 跳躍線偏移、速度、觸控偏移、翻譜器模式 |
| Stamp Sizes | 各標記符號的個別尺寸調整 |

**System Settings 重要設定：**

| 設定 | 說明 | 預設值 |
|---|---|---|
| Jump Offset | 跳躍線距視窗頂部距離（px） | 40px |
| Jump Speed | 跳躍動畫時間（ms），數值越小越快 | 300ms |

### 8.2 標記工具設定（Stamp Settings）

```
標記工具（T）→ 設定圖示
```

包含四個分頁：

#### Size（大小）

| 設定 | 說明 |
|---|---|
| Scale | 此份樂譜的標記縮放比例（0.5–3.0x） |
| Font Size | 文字標記的字體大小（16–32px） |

#### Touch（觸控）

| 設定 | 說明 |
|---|---|
| Offset Y | 觸控點擊時，標記相對手指的垂直偏移 |
| Offset X | 觸控點擊時，標記相對手指的水平偏移 |

#### Categories（類別）

顯示各圖層的可見性切換與批量清除。

#### More（進階）

| 設定 | 說明 | 預設值 |
|---|---|---|
| **頁面重疊** | Jump 時，以倒數第 N 個 System 為起點 | 1 |
| System Detection | 開關顯示 + 手動偵測 / 清除 | — |

## 9. 標記分享（Gist Share）

Gist Share 讓你把標記（弓法、指法、詮釋記號等）透過一個 URL 傳給任何人，對方開啟連結後即可在自己的 ScoreFlow 看到完整標記，並能繼續自行編輯。

> **PDF 不含在連結中。** 接收方需自備同一份 PDF 檔案。分享內容只包含標記資料與樂譜識別碼（fingerprint）。

### 9.1 誰需要 GitHub 帳號？

| 角色 | 需要帳號 | 說明 |
|---|---|---|
| **分享方** | ✅ 需要 GitHub 帳號（一次性設定） | 建立 GitHub Personal Access Token，上傳 Gist |
| **接收方** | ❌ 不需要任何帳號 | 直接點開連結即可 |

### 9.2 如何分享（分享方）

1. 開啟任一樂譜，確認有標記。
2. 點擊 Doc Bar 右上角的**分享按鈕**（上傳箭頭圖示）。
3. **首次使用：** 系統會要求連結 GitHub 帳號：
   - 前往 `github.com/settings/tokens/new`
   - 名稱隨意填，勾選 **gist** 權限
   - 產生後複製 token，貼入 ScoreFlow 的輸入框
   - 完成後 token 存入本機，下次分享不需重複設定
4. 若有**斗篷標籤（Cloak Labels）**，系統會詢問要包含哪些斗篷群組（預設全勾選）。
5. 上傳完成後，顯示分享連結——**自動複製到剪貼簿**，亦可在對話框中選取複製。
6. 將連結傳給對方即可。

### 9.3 如何接收（接收方）

**情況 A：你的裝置已有此份樂譜**

1. 點開分享連結，ScoreFlow 自動偵測並下載標記資料。
2. 顯示對話框，說明來自誰、包含幾個標記，並詢問：
   - **新增詮釋風格** → 以新的「詮釋風格（Persona）」加入，與你自己的標記並存，可獨立顯示 / 隱藏
   - **覆蓋目前標記** → 取代此份樂譜的所有標記（⚠️ 不可復原，建議先備份）
3. 選擇後，標記自動套用。

**情況 B：你的裝置尚未有此份樂譜**

1. 點開連結後，系統顯示對話框並提示選擇 PDF 檔案。
2. 選擇對應的 PDF → ScoreFlow 計算 fingerprint 並確認配對。
3. 標記套用，PDF 存入本機（下次開啟免選）。

### 9.4 管理分享連結

| 事項 | 說明 |
|---|---|
| **撤銷連結** | 前往 `github.com/gists`，手動刪除對應的 Gist |
| **連結永久有效** | 只要 Gist 未刪除，連結可無限次使用 |
| **每次分享產生新 Gist** | 舊連結持續有效，不會被新連結覆蓋 |
| **登出 GitHub** | 設定 → 分享設定 → 登出 GitHub（清除 token） |

### 9.5 注意事項

- Gist 為 **Private（unlisted）**：不會出現在搜尋結果，但知道連結的任何人都能讀取，並非加密。
- 標記資料中**不包含** PDF 內容，不涉及版權問題。
- 分享連結只包含**使用者標記**（弓法、指法、Anchor 等），不包含自動偵測的五線譜行資料（system stamps）。
- 需要網路才能上傳 / 下載，離線時會顯示錯誤訊息。

## 10. PWA 安裝與離線使用

ScoreFlow 是一款 **PWA (Progressive Web App)**，這意味著它可以像原生應用程式一樣安裝到您的桌面上或手機主畫面上，並且在完全沒有網路（例如演藝廳、飛機上）的情況下仍能正常運作。

### 10.1 為什麼要安裝成 App？

- **全螢幕體驗**：移除瀏覽器網址列，將所有視窗空間留給樂譜。
- **離線啟動**：即使沒有 Wi-Fi，也能從主畫面直接開啟 App 並讀取樂譜。
- **快取優先**：PDF.js 渲染引擎與相關組件會被永久快取在您的設備中，提升啟動速度。

### 10.2 安裝指南

根據您的裝置與瀏覽器，安裝方式略有不同：

#### **iOS / iPadOS (Safari)**
1. 在 Safari 中開啟 ScoreFlow 網址。
2. 點擊工具列底部的 **分享 (Share)** 圖示（箭頭向上方框）。
3. 向上捲動選單，點擊 **加入主畫面 (Add to Home Screen)**。
4. 確認名稱後點擊右上角 **新增**。現在您可以從桌面開啟 ScoreFlow。

#### **Android / Chrome / Windows / macOS**
1. 開啟 ScoreFlow 後，觀察網址列右側。
2. 如果出現 **螢幕下載圖示** 或 **「安裝」** 字樣，直接點選。
3. 若未出現，點擊瀏覽器右上角的 **三個點 (⋮)**。
4. 選擇 **安裝 ScoreFlow (Install)** 或 **將專頁加入桌面應用程式**。

#### **Edge (Windows/macOS)**
1. 開啟 ScoreFlow 後，點擊網址列右端的 **應用程式可用圖示**（三個正方形加一個加號）。
2. 點擊 **安裝**。
3. 或者點擊 `...` → **應用程式** → **將此站點安裝為應用程式**。

### 10.3 離線使用注意事項

1. **首次快取**：建議在有網路時至少開啟一次 ScoreFlow 並載入您的常用樂譜，確保所有資源（含 PDF.js Worker）都已存入快取。
2. **數據持久性**：ScoreFlow 將樂譜與標記儲存在瀏覽器的 **IndexedDB** 中。**請勿**清除瀏覽器的「網站資料」或「快取」，否則本機樂譜需重新從檔案選取或從雲端載入。
3. **PWA 存取限制**：在 iOS 上，PWA 軟體與 Safari 瀏覽器共用存儲空間，但它們是獨立的執行環境。若您在 Safari 設定了某些東西，在 PWA 版本可能需要重新載入以同步狀態。
4. **Service Worker 更新**：當開發者釋出新版本時，系統會在您下次連網開啟時在背景下載更新，並在您下一次啟動時自動套用新版。
 
### 10.4 軟體更新機制

ScoreFlow 採用自動化更新機制，確保您始終使用最新且最穩定的版本：

- **自動檢查**：
  - **啟動時**：每次您在連網狀態下開啟或重新整理 App，系統會立即檢查有無新版本。
  - **背景檢查**：若 App 長時間保持開啟，瀏覽器通常會每隔 **24 小時** 自動於背景檢查一次。
- **自動安裝**：一旦偵測到新版本，系統會於背景自動下載並安裝。新版本通常會在您**下一次啟動 App** 或重新整理後立即生效，無須手動操作。
- **版本確認**：
  - 您可以前往 **Global Settings -> System Settings** 查看最下方的 **Build Info**。
  - **Build Time**：顯示該版本的最後編譯時間，可用於確認是否已更新至最新狀態。
