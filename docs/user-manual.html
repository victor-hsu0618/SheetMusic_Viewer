Section 01

快速開始

三步驟讓你立刻開始使用 ScoreFlow。

  • 1

    匯入樂譜 PDF

    點擊「+ Score」按鈕或拖曳 PDF 至視窗。系統計算 SHA-256 指紋，確保標記與正確樂譜版本綁定。

  • 2

    自動五線譜偵測

    PDF 載入後，系統在背景自動分析每頁像素，找出所有樂譜行（System）的位置，供 Jump 導航使用。無需任何設定。

  • 3

    按 Space 翻頁

    直接按 Space 或 → 跳至下一個樂譜行。系統依序以 Anchor → System → Fallback 三段邏輯決定落點。

  • 4

    （可選）放置 Anchor 精準控制

    按 A 鍵進入 Anchor 模式，在需要精準停靠的位置點擊。Anchor 優先於自動偵測的 System。

💡 PWA 離線支援

安裝到 iPad 主畫面後，即使沒有 WiFi（音樂廳、排練室）也能完全正常使用。

Section 03

Jump 導航系統

ScoreFlow 最核心的功能，專為演奏中或練習時快速翻頁設計。

3.1 跳躍線（Jump Line）

視窗中有一條水平虛線，稱為跳躍線。它定義了視線焦點的位置——Jump 後，目標內容會對齊在這條線上。

屬性說明
預設位置距視窗頂部 40px
調整方式Global Settings → System Settings → Jump Offset
視覺提示水平虛線（可開關），尺規橘色標記指示下一目標

3.2 Jump 前進邏輯（按 Space 或 →）

系統依以下優先順序決定跳躍目標（連續捲動模式下會有額外的頁跳邏輯，此處為標準模式）：

1

Anchor 存在

跳至跳躍線下方最近的 Anchor（精準控制）。尺規上橘色標記指示目標位置。

2

System 存在（五線譜自動偵測）

依「頁面重疊」設定，取畫面中倒數第 N 個可見 System，跳至它對齊跳躍線的位置。詳見 3.3。

3

最終 Fallback

捲動 (視窗高度 − 跳躍線偏移量)，使原畫面底部對齊新畫面的跳躍線。

3.3 頁面重疊（Page Overlap）

這是 System-based 跳躍的核心設定。翻譜時，你希望「保留畫面中倒數第 N 個 System 作為過渡銜接」——它會出現在新畫面的跳躍線位置。

跳躍前
── 跳躍線 ──────────────
System B
System C
System D　←── 倒數第 1
→
跳躍後（N = 1）
── 跳躍線 ──────────────
System D　←── 對齊跳躍線
System E
System F

System D 是原畫面最後一行，跳躍後它出現在新畫面的跳躍線上，作為翻頁銜接。

設定值 N效果
1（預設）最後一個 System 對齊跳躍線 → 最大跳躍距離
2倒數第二個 System 對齊跳躍線 → 多保留一行上下文
3+保留更多銜接上下文，跳躍距離更短

設定路徑：標記工具（T）→ 設定 → More → 頁面重疊 [−] N [+]

3.4 Jump 後退（按 ←）

每次 Jump 前進都會記錄位置到歷史堆疊（最多 50 筆）。按 ← 精確回到上一個位置，與跳躍距離無關。

若歷史堆疊為空（剛開啟樂譜後的第一次後退），會往上捲動一個等距。

Section 04

五線譜行偵測

Staff Detection 透過畫面像素分析，自動找出每一頁的樂譜行（System）位置，讓 Jump 在沒有手動 Anchor 的情況下也能精準導航。

4.1 自動偵測

開啟任何新 PDF 後，系統在背景自動執行偵測，並在畫面顯示進度提示：

正在分析樂譜結構... 2 / 8

偵測完成後，結果儲存於本機（IndexedDB），下次開啟同一份 PDF 不會重複偵測。

📄 支援掃描版 PDF

演算法採用自適應背景門檻，取樣頁面邊緣估算背景亮度，可正確處理泛黃、陰影等掃描背景。

4.2 在尺規上顯示 System 位置

前往 標記工具（T）→ 設定 → More → System Detection，開啟最上方的顯示開關：

  • 開啟 → 尺規上顯示淡藍色虛線，標示每個 System 的頂部位置
  • 其中「下一個 Jump 目標」以橘色顯示（光暈效果）
  • 預設關閉（不影響導航，純視覺輔助）

4.3 手動重新偵測

若偵測結果不準確（例如特殊版面），可手動重新執行：

按鈕說明
Detect清除舊結果，對目前 PDF 重新偵測全部頁面
刪除全部清除所有 System 標記，Jump 退回 Fallback 模式

路徑：標記工具（T）→ 設定 → More → System Detection

4.4 偵測演算法參數

偵測結果以比率（ratio）儲存（System 頂部 Y ÷ 頁面高度），與縮放比例無關。

自適應背景門檻
邊緣 6 區塊取樣 × 0.75
估算頁面背景亮度，避免掃描版誤判深色背景。
行密度門檻
≥ 30% 橫向像素
超過此比例的橫向深色像素，才認定為樂譜線行。
最大線條厚度
≤ 8px
超過 8px 的深色帶視為文字或樑線（beam），排除。
System 分群間距
> 25px
線條群組間距超過此值，認定為不同的 System。
Section 05

Anchor 錨點

Anchor 是使用者手動放置的跳躍目標，優先順序高於 System 自動偵測。

5.1 放置 Anchor

  1. 按 A 鍵，或點擊工具列的錨點按鈕，進入 Anchor 模式
  2. 點擊樂譜中的任意位置，放置錨點標記（尺規上顯示藍色短橫條）
  3. 再按 A 或 V 退出 Anchor 模式

5.2 Anchor vs System 的使用情境

情境建議
一般練習 / 表演瀏覽使用 System（自動，零設定）
需要在特定段落精準停靠放置 Anchor（手動，完全控制）
混用Anchor 優先；Anchor 用完後自動切換至 System
💡 尺規橘色標記

尺規上的橘色標記始終指向「下次按 Space 的落點」——無論是 Anchor 還是 System，視覺反饋完全一致。

專為職業樂手設計的分層標記架構。

6.1 開啟標記工具

按 T 或點擊工具列的 T 按鈕，開啟標記面板。

6.2 標記類別

6.3 Edit 工具列

標記面板最上方的 Edit 工具列提供對已放置標記的操作功能。

工具功能操作方式
Select V 選取並拖曳移動標記 點擊標記後拖曳到新位置；放開後位置儲存
Copy 複製標記 點擊標記複製一份，複製品自動進入拖曳模式，拖到目標位置放開
Cycle 輪轉標記類型 點擊標記，自動切換到同組的下一個類型（見下方說明）
Eraser E 刪除單個標記 點擊欲刪除的標記；靠近時顯示紅色陰影作為提示
Cycle 工具 — 支援的輪轉組合
類型輪轉順序
弓法上弓（Up Bow）↔ 下弓（Down Bow）
指法拇指 → 1 → 2 → 3 → 4 → 5 → 拇指（循環）

靠近標記時顯示藍色陰影，表示可點擊輪轉。每次點擊前進一格。

💡 Select / Cycle 的懸停提示

使用 Select、Copy 或 Cycle 工具，將游標移近標記時，標記會出現藍色發光效果，代表已定位到該標記。觸控裝置上直接點擊位置即可。

6.4 小節數工具（Measure Number）

在樂譜左緣放置小節編號標籤，方便排練時快速對齊小節位置。小節數標籤會同時出現在樂譜上與左側尺規。

  • 1

    選取工具

    在標記面板（T）選取小節數工具，或從 Anchor 類別中找到「Measure」。

  • 2

    設定起始編號與步距

    點擊樂譜，系統彈出數字鍵盤：輸入此處的起始小節號，並設定自動遞增步距（通常為 1）。確認後標籤放置完成。

  • 3

    繼續放置

    之後每次點擊，編號自動依步距遞增（例如 1、5、9…）。無需重新輸入。

  • 4

    尺規上拖曳調整位置

    尺規上的小節數標籤支援上下拖曳——直接在尺規上拖動標籤到正確的行，位置即時同步更新至樂譜。

💡 標籤固定在左緣

小節數標籤的水平位置自動鎖定在樂譜左緣，確保所有編號整齊對齊，不需手動調整 X 軸。

6.5 圖層管理

每個類別可獨立顯示 / 隱藏：標記工具（T）→ 設定 → Categories。

Section 09

匯出與備份

將標記資料備份、分享給他人，或輸出含標記的合併 PDF。

入口：點擊 Score Detail 面板（側邊欄或 Doc Bar 中的樂譜名稱）→ System 分頁 → Score Annotation Exchange 區塊。

9.1 匯出標記資料（Export Data）

將目前樂譜的所有標記（stamps）、圖層設定、來源資訊匯出為 .json 檔案。

包含內容說明
Stamps所有 Anchor、Measure、Pen、Bowing、Fingering 等標記的位置與資料
Layers圖層顯示狀態與設定
Sources協作來源（Persona）資訊
Metadata樂譜名稱、作者、匯出時間戳
💡 定期備份

標記資料儲存於本機 IndexedDB。清除瀏覽器資料或更換設備時，請務必先 Export Data 備份。

9.2 匯入標記資料（Import Data）

載入 .json 備份檔，系統會詢問匯入方式：

選項行為適用情境
Merge as New Persona將匯入的標記以新來源（新 Persona）加入，與現有標記並存合併他人的標記、接收樂團同伴的詮釋
Overwrite Current以匯入資料完全覆蓋目前的所有標記從備份還原、換設備後恢復資料
⚠️ Overwrite 無法還原

選擇 Overwrite 前，建議先 Export Data 備份現有標記，以防需要回復。

9.3 匯出合併 PDF（Export Merged PDF）

將所有可見標記直接燒錄進 PDF，輸出一份含標記的獨立 PDF 檔。

特性說明
標記渲染所有可見圖層的標記（Pen、Bowing、Fingering 等）合併至 PDF 頁面
Anchor 排除Anchor 錨點為純導航用途，不會出現在匯出的 PDF 中
用途列印樂譜、與不使用 ScoreFlow 的合奏夥伴分享含標記的譜面
